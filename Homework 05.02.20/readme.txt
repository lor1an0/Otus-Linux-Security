1) Для того, чтобы разрешить только пользователю otus монтировать диски, было написано разрешающее правило для policykit 10-mount.rules. В правиле происходит сопоставление группы пользователя и совершаемого действия, и в случае совпадения монтирование разрешается.
Для реализации необходим пакет udisks2

2) Для разрешения пользователю otus2 подключаться по SSH только с 12 до 20 часов каждый день использовался pam-модуль pam_time.so. Соответственно, строка account required pam_time.so была добавлена в цепочку pam для sshd.

3) Для смены рута пользователю otus3 использовался параметр sshd ChrootDirectory. Соответственно, в домашней папке пользователя был создан отдельный корневой каталог chroot, в который был скопирован минимальный набор библиотек и команд для подключения и работы.

Также был установлен пакет policycoreutils-python для управлением контекстом SELinux. Без этого при копировании файлов time.conf, sshd_config и 10-mount.rules копировался базовый контекст файла, а не наследовался контекст директории копирования. Из-за этого sshd, pam и policykit не могли получить доступ к этим файлам.
